# Contributing to Slipstream

First off, thank you for taking the time to contribute — it genuinely helps.

## Contributions

Contributions are welcome in many forms, including but not limited to:

- Bug reports
- Feature proposals
- Documentation improvements
- Code contributions
- Design or process feedback

Please keep contributions focused, well-scoped, and documented where appropriate.

## Expectations

- Be respectful and constructive
- Favor clarity over cleverness
- Follow existing project conventions
- When in doubt, open an issue or discussion first

## Questions

If you're unsure about anything, feel free to open an issue or start a discussion — clarity beats assumptions.
